Project

Agent Client Protocol

我关心的不是概念名词，而是 Agent、工具、环境、子代理之间到底怎么协作，哪些接口会决定一条工作流能不能稳定下来。

我为什么会看这个

很多 Agent 展示都很酷，但一旦进入真实任务，就会遇到上下文切换、工具权限、状态同步、子代理协作这些问题。协议层和边界设计，决定了它是 demo，还是系统。

我更想研究的是，如何让这些能力变成长期可复用的基础设施，而不是一次性惊艳演示。

现在的关注点
  • Agent 如何调用工具并保持结果可验证
  • 子代理如何分工，而不是只堆上下文
  • 协议层怎么服务本地工作流和真实项目
状态

研究中。重点不是先得出最终结论，而是先做足够多的真实使用观察。

我想避免的误区
  • 只看概念，不看执行细节
  • 只看单点能力，不看整条链路
  • 只追新名词，不追真实可用性

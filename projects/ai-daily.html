Project · 自动化工具

AI 日报

每天自动从全球 AI 媒体抓取资讯，本地 LLM 生成中文摘要，输出一份可直接阅读的日报。

为什么做这个

AI 领域信息量太大，每天在各个网站手动刷信息效率很低。我想要的是：每天早上起来，一份整理好的中文摘要已经在那里，而不是我再去各处拼凑。

这个项目分两条线：一条是 Antigravity 技能，直接在对话里召唤，用 web 抓取和搜索即时生成今日简报；另一条是本地 Python 工具，定期跑 RSS 抓取 + Ollama 推理，生成 Markdown 和 HTML 两种格式的离线日报。

两种实现路径
  • Antigravity 技能版：基于 daily-ai-news skill，调用 web reader 实时抓取 VentureBeat、TechCrunch 等主流 AI 媒体，四阶段工作流（抓取 → 过滤 → 分类 → 输出），对话即用，零本地依赖
  • Python 本地版：RSS 源 7 个（Google AI Blog、MIT Tech Review、Hugging Face、arXiv cs.AI 等），feedparser 解析，qwen2.5:3b 本地生成中文摘要，输出 .md + .html 双格式离线报告
  • 容错设计：每个 RSS 源独立超时处理，失效就跳过，整体不崩；Ollama 未启动时摘要位置写占位提示，不影响报告生成
技术组成
  • Antigravity（OpenClaw）daily-ai-news skill —— 实时在线版入口
  • Python + feedparser + requests —— RSS 抓取与解析
  • Ollama qwen2.5:3b —— 本地推理，生成中文摘要，无需 API 费用
  • HTML 报告：玻璃态卡片设计，渐变背景，支持深色阅读
我真正想解决的问题

信息过载不是信息太多，而是没有筛选层。我的目标不是收集更多，而是每天花最少时间，拿到最值得看的那 10 条。

现在这套流程跑通之后，我已经不再每天手动刷 AI 资讯了。日报生成了，我扫一遍，发现有意思的再点开原文。这个边界很重要——工具帮我拦截信息噪声，不帮我做判断。

状态

两条线都已跑通。Antigravity 版日常在用，Python 版作为离线备份和格式化输出。

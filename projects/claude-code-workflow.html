Claude Code 工作流配置

我以后说 cc，默认就是 Claude Code。这个项目页记录的，不是它能做什么，而是我怎么让它真的进入我的开发、整理和写作动作里。

为什么值得单独写

因为很多工具介绍都停在能力列表，但真正有价值的是它怎么被用进现实任务。比如 specs 怎么写、什么时候让它先做设计、什么时候直接让它改代码、什么时候要回到我自己做判断。

对我来说，cc 最重要的价值不是代替思考，而是帮我把一个明确任务推进得更快、更稳。

现在的做法
  • 先写规格和边界，再让它执行
  • 优先让它做明确、可验证的小任务
  • 保留我自己的判断，不把输出直接当结论

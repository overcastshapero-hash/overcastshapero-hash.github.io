阶段复盘 2026.04.15

AI 学习第一阶段结束时，我终于知道自己真正需要什么

真正的变化，不是我又学会了多少个工具，而是我终于开始从“AI 玩家”往“AI 系统使用者”转变。

从年初的 openclaw 热潮开始，到后来接触 Codex、Claude Code，再到现在把 Hermes 变成主力，我越来越觉得，AI 这件事已经不是“会不会问问题”那么简单了。

如果一定要给我现在的状态下一个判断，我会说：我已经过了新手体验期，正处在“重度探索者”走向“初级系统使用者”的阶段。

这一阶段，我接触到的早就不只是模型

我接触到的东西已经是一整套环境：API key、skill、MCP、hooks、本地 agent、自动化、记忆、工作流、知识库，还有不同模型和不同工具之间的切换与配合。

从 openclaw 开始，我第一次真正意识到，AI 不只是一个会回答问题的聊天框，它开始长出手了。它可以调用工具，可以执行动作，可以被接入本地环境，可以成为一个真正意义上的助手。

也正是从那个阶段开始，我开始频繁接触 GitHub 仓库、各种配置文件、各种 API key，开始理解 AI 不只是产品，更是一种可以进入个人系统的能力。

我体验了很多工具，但最后留下来的并不多

Cursor 是我整体用下来最舒服的，Antigravity 做出的网站效果其实不错，Claude Code 目前还是我体验里最强的一个，Codex 的优势则在于稳定、量大、能扛。

但如果对自己诚实一点，我最后真正愿意长期留住的，不是“某个点最强”的工具，而是能进我日常系统的工具。

也正因为这样，我最后把 Hermes 变成了自己的主力。它最吸引我的，不是某一项单点能力最强，而是它在“个人系统”这件事上最完整：自我进化、自动化、定时任务、外接接口、skill、保存记忆，这些能力一叠加，它就不再只是一个工具，而更像是一个可以长期协作的个人助理。

但我也被一个现实反复打醒

我最大的问题，不是没热情，而是太容易被新鲜感带着走。

X 上有很多东西，YouTube 上也有很多东西，新的模型、提示词、产品、工作流总在冒出来。我确实在上面学到了不少，但我也越来越清楚地感觉到，自己很容易陷入一种循环：看到新的，立刻尝试，体验到新鲜感，然后很快不用，再去追下一个。

这让我越来越确定：我现在不是缺信息，而是缺把信息真正转化为沉淀和能力的机制。

这也是我为什么越来越反感“工具崇拜”

AI 世界里最容易让人上头的，就是你会一直觉得：是不是还有一个更强的模型、一个更好的 agent、一个更完整的 workflow，只要我找到它，我就彻底打开了。

但后来我发现，这种想法本身就有问题。很多时候你不是在寻找更好的工具，而是在给自己继续不开始找理由。

我现在越来越倾向于用几个问题筛工具：

  • 它稳不稳定？
  • 它贵不贵？
  • 它能不能进入我的真实生活？
  • 它能不能帮我减少重复劳动？
  • 它能不能形成长期可复用的系统？

如果答案都不够好，那它就算再强，也未必适合我。

这段时间我也开始做自己的东西了

这阶段最让我高兴的地方，不只是我用了多少工具，而是我开始做出一些真正属于自己的东西。

我把自己蒸馏成了一个 skill，虽然现在它还更像一个拿来逼自己行动的外部约束；我做了个人网站，做了番茄钟，还做了一个无用网站。更重要的是，我开始正式构建自己的 Obsidian 知识库，并决定把这套东西长期用下去。

这说明我已经不满足于“用别人的工具”，而是开始想做自己的系统，想让 AI 真正进入我的学习、记录、输出、生活管理，甚至未来的表达方式里。

我现在最清楚的一点是：下一阶段不是继续扩张，而是收缩

这一阶段结束之后，我最大的收获不是学会了多少工具，而是终于知道自己真正需要什么样的 AI 系统。

我接下来最该做的，不是继续无止境地找更新、更强、更新鲜的工具，而是把已经得到的这些能力、经验、判断和资产，真正沉淀成属于我自己的系统。

换句话说，不是继续扩张，而是收缩：

  • 收缩工具栈
  • 固定主力工具
  • 建立稳定 workflow
  • 让 AI 真正进入记录、复盘、输出和日常决策

只有这样，我这一路上接触过的东西，才不会只是一次又一次的新鲜体验，而会真正变成未来几年持续受益的底层能力。

最后

如果要用一句话总结我这一阶段，我会说：

我最重要的成长，不是学会了多少个 AI 工具，而是开始知道，自己真正需要的是什么样的 AI 系统。

这听起来没有“我已经掌握了 AI”那么热血，但它更真实。因为我知道，真正的门槛从来不是尝鲜，而是沉淀；不是接触，而是进入生活；不是兴奋，而是形成闭环。
